Operator control panel · mobile design study

a2a-bridge — Mobile, reframed

Not a pager — a primary interface. At work: laptop + TUI. At home: this app, over the operator's existing Tailscale mesh (the same authenticated transport that already carries their mosh sessions). The centerpiece is conversing with agents — reading transcripts the way a good editor renders them, and replying with a real composer — because that's the daily act a phone terminal makes merely tolerable.

  • failed · tier-3 quarantine

Transport

Tailscale is the tunnel. The bridge stays bound to the tailnet interface; mesh ACLs limit it to the operator's own devices. The app speaks the same seams as the TUI — A2A JSON-RPC, SSE with Last-Event-ID reattach, the [traces] routes — through the shared bridge-client payload builders, so it can never send anything the CLI couldn't. Per-device tokens (H3-2) layer on later; they don't gate this.

02

Agents — fleet + config editing

read: now · write: admin sidecar

The [[agents]] registry as cards — kind, sandbox tier, pinned model/effort, MCP wiring, doctor health — and a real mobile form to add or edit an entry. The registry's mutation surface is deliberately the config file + hot-reload (ADR-0005); there is no remote admin API, and none gets bolted onto serve. Writes go through a tiny mesh-only admin sidecar on the host that runs validate against a temp copy, then atomic-renames into place — the bridge's own 200 ms hot-reload is the apply mechanism.

03

Workflows — run now, edit honestly

run: now · edit: sidecar + restart

List [[workflows]], see the DAG, run one against a repo — running is pure existing surface (submit <workflow> --input, then the conversation view watches it). Editing is where honesty matters twice: definitions aren't served over the wire (the sidecar reads the config), and workflows are read once at boot — an edit needs a serve restart, which the durable store + crash-resume makes survivable. The builder is a node list + per-node form, not a canvas: on a phone, dragging edges is a party trick; picking inputs from a list is engineering.

Same discipline as the TUI: every screen is a client over a named seam — A2A JSON-RPC + SSE, the [traces] routes, the Agent Card — plus one honest addition, a mesh-bound admin sidecar that turns the config file (the registry's real mutation surface, ADR-0005) into a validated, atomic write-back path.
