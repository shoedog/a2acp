Agents/Workflows refactor — the two shifts, visually

Companion to the RFC. Two changes: split the fused AgentEntry into a composable Agent over a Provider, and make workflow definitions runtime-mutable — both landing without disturbing the runtime, because the registry already swaps atomically and runs already pin their graph by value.

01

The Agent / Provider split

Today one flat struct fuses execution substrate, model tuning, tools, and identity — and carries no role (role lives only in the node prompt). The target splits ProviderEntry (adapter/sandbox/watchdog/model) from AgentDef (role-prompt/tools/model-defaults), composing them into exactly today's AgentEntry above the registry — so bridge-registry, bridge-acp, and the executor never change.

02

Runtime workflows — apply flow & run immutability

One file, one parse, ordered apply: registry then catalog, so a workflow can never publish referencing an agent the published registry lacks — stronger than today's one-time boot check. And the scary part is already solved: a run pins its graph by value into the task store, so in-flight and resumable runs are immune to edits.

Phase 0 bug

Found while grounding the RFC and verified: registry.rs:496-511's warm-reuse criterion omits the spawn-frozen mcp/mcp_delivery/watchdog, so editing only an agent's tools hot-reloads into a stale warm process. Filed as issue #35 — a prerequisite fix for the split, where agents are differentiated chiefly by toolset.

03

Phased plan

  • Phase 0Reuse-criterion fix (issue #35) — add mcp/mcp_delivery/watchdog to the spawn-frozen comparison. Small, ships alone, prerequisite for both shifts.
  • W1WorkflowCatalog + unified snapshot + ordered reconcile + def_hash. The cheapest first increment; executor untouched.
  • W2WorkflowConfigStore write-back + Coordinator/MCP workflow upsert/remove/list — the DAG editor's server half.
  • S2aDomain split, pure refactor — ProviderEntry/AgentDef/compose_agent + legacy synthesis; golden parity across examples/.
  • S2bNew capability — [[providers]] syntax, role_prompt composition, node overrides, three-layer effective_config.
  • S2cOptional, evidence-gated — provider slot-sharing by spawn-frozen InstanceKey; per-agent AgentCard skills.
The bright line, preserved from ADR-0008: agents are configuration, workflows are data, execution is ports — the bridge never plans, and the orchestrator remains the caller. Full RFC with code refs: rfc-agents-workflows.md.
