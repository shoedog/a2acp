## Goal
Give the orchestration core a **stable Rust service API (the `Coordinator`)** and a **stdio MCP adapter**
over it, so a coding-agent orchestrator can drive the bridge (run a workflow, prompt/continue a warm
agent, check status, clear, cancel) **as a native MCP tool — no Bash shell-out**. A2A, the CLI, and MCP
become **co-equal thin adapters over the one Coordinator** (built FIRST; the surfaces call it).
**D1:** operation params (workflow, input, context, agent/model/effort/mode/cwd) are **typed fields**
populated identically from MCP `arguments` / CLI flags / A2A `a2a-bridge.*` metadata — killing the need
for per-role override `.toml` copies (for LOCAL prompt/continue; workflow-node overrides deferred).

## KEYSTONE constraints (do not violate — converged dual-lens)
1. **ONE Coordinator owns the state; surfaces are thin adapters.** A2A-HTTP, in-process CLI, and
   MCP-stdio each call the SAME `Coordinator` methods. NOT "CLI/MCP thin over the A2A JSON-RPC wire."
   Build the Coordinator FIRST.
2. **Single-owner durable store.** `SqliteStore::open` holds an exclusive cross-process `<path>.lock`
   (`bridge-store/src/sqlite.rs:37-64`). The `mcp` process is a STANDALONE single-owner: it takes the
   same lock and **fails clearly if a serve already owns it**. NO two-process sharing of one sqlite.
   Cross-process WARM-session sharing is OUT (SessionManager is a process-local `HashMap`
   `session_manager.rs:116-124`); the composite `mcp --http` that delivers it is DEFERRED (D-B').
3. **Two distinct lifecycles, two tools.** `run_workflow` = detached multi-node submit → `task_id`
   (reuses `spawn_detached_workflow` + the W3a/W3b durable substrate). `prompt`/`continue` = a warm
   single-turn against a `ContextId`, returning ONE collected text result (the `unary_message` collect
   shape, `server.rs:2842`). Do NOT collapse them.
4. **stdout is RESERVED for framed NDJSON replies.** The `mcp` subcommand MUST route tracing to
   stderr/file (NOT the `bridge_observ::init()` default-stdout, `bridge-observ/src/lib.rs:5-10`) and emit
   NO `println!` on the Coordinator path. Any stdout write corrupts the stream.
5. **Reuse the lsp-mcp protocol shell VERBATIM.** Lifecycle handshake (`initialize` ECHOES the client
   `protocolVersion` — NOT the A2A version-REJECT policy), `tools/list`, `tools/call` gated on
   initialized, the `result.content[0].text` ok / `result.isError` envelopes, the `-32600/-32601` codes
   (`lsp-mcp/src/mcp/{transport.rs,mod.rs,error.rs}`). Port the I/O to async (tokio) + a serialized
   writer; reuse `bridge-acp/src/framing.rs`'s size-limited async NDJSON reader.
6. **Pure, suite-gated move.** The Coordinator extraction (the crate move) changes NO behavior — every
   moved object keeps its semantics; the full workspace suite is the gate after each move step.

## DoD / gate
- **Live-gate (real codex):** `a2a-bridge mcp --config <livegate>` driven by a scripted MCP stdio client
  (NDJSON): `initialize` → `tools/list` shows the tools → `run_workflow` returns a `task_id` →
  `status {task_id}` shows it progress to terminal → `run`/`continue` on a `context` recalls a codeword
  across two turns (warm) → `clear` resets → `cancel_task` flips a running task. NO Bash shell-out; all
  via MCP tools.
- **Identical state (per FIX-4 — SPLIT gate; the CLI `session`/`task` cmds are HTTP clients of a serve,
  not direct store readers `main.rs:2994`):** durable TASK state is checked via a same-process surface OR
  `task get` against the store AFTER the mcp process exits; warm SESSION state is checked via MCP `status`
  BEFORE EOF (or a same-process harness). Do NOT require post-exit `session status` equivalence (no serve
  → nothing to query; single-owner → no concurrent reader).
- **Single-owner:** `a2a-bridge mcp` while a serve holds the store → a clear error + nonzero exit (no
  corruption).
- **Non-divergence test:** a unit/integration test asserts the migrated A2A handlers + CLI + MCP dispatch
  each call the SAME `Coordinator` method (the route logic exists in exactly one place) and share ONE
  `Arc<dyn TaskStore>`.
- **Stdout discipline:** a test (or the live-gate) asserts the `mcp` stdout carries ONLY framed NDJSON
  (no tracing/`println!` leakage).
- **Gate:** fmt + clippy `--all-targets` clean; full workspace suite green; the crate move is behavior-
  preserving (the pre-existing suite passes unchanged).
