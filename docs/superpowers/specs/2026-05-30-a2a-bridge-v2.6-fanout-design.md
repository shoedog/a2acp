# A2A Bridge — Increment 2.6 Design Spec (Fan-out + Concurrent Stream-Merge)

*Date: 2026-05-30 · Revised after Codex + Claude spec review*
*Status: Approved design (revised) — ready for implementation planning*
*Builds on: v1 + Increment 2.5 (both merged to `main`)*

---

## 1. Purpose

Add a **`fan-out` skill**: one inbound A2A task runs the same prompt on **both the local Kiro
agent and the configured remote A2A peer** concurrently; the bridge **merges their event
streams into the single caller SSE, source-labeled, both running to completion** (a "second
opinion"). The merge coordinator is shaped **N-ary** (a `Vec` of labeled sources; 2.6 wires
two) so Increment 3's multi-agent backends generalize without a refactor.

This realizes the concurrent stream-merge deferred from 2.5 §8. Review tags: `[Cx]`/`[Cl]`.

## 2. Decisions (locked, incl. review revisions)

| Decision | Choice |
|----------|--------|
| Semantics | Second opinion: same prompt to both sources; both run to completion; both results returned, source-labeled. Not race-for-fastest. |
| Sources | Kiro (local) + the one configured `[delegation]` peer; coordinator is **N-ary** (`Vec<Source>`), 2.6 passes two. |
| Source label | `Event.source: Option<String>` (a source **id**, `"kiro"`/`"peer"`); **no `Local\|Peer` enum** on core `Event` `[Cl3]`. |
| Labeling | Per-frame `metadata["a2a-bridge.source"]` on every `StreamResponse`; artifact frames also set `artifact.name`. |
| **Coordinator owns the streams** | `fanout.rs` builds `Translator::run(...)` + `Delegation.events` **directly** and is the **sole sender** to the `mpsc`. NOT "two independent producers into one mpsc" (which cannot do degrade/both-failed) `[Cx2/Cl2]`. |
| **Task-terminal signal** | The task ends with a final `StatusUpdate(Completed\|Failed)`. Per-source artifacts use `lastChunk` for **artifact**-completion only, NOT task end. **The 2.5 outbound parser is updated** to end on the terminal status and accumulate multiple artifacts `[Cl4 / user decision]`. |
| Degrade policy | Keyed on `BridgeError::disposition()`: only `SetState(Failed)`-disposition errors degrade a source to a labeled error and continue the survivors. A `RejectRequest`/resumable (`InputRequired`/`AuthRequired`) error on a fan-out source is **treated as that source failing** (labeled error) — fan-out does NOT suspend per-source `[Cx4]`. Both/all sources failed → task `Failed`. |
| Per-source startup errors | `delegate()`/`open_stream()` can fail **before** a stream exists → that source starts in the failed state (labeled error), survivors continue `[Cx3]`. |
| Cancel | Cancel **all** sources. Cancel Kiro **immediately** by its known `SessionId` (do NOT await the peer id); latch the peer cancel and apply when the id appears. Guard keyed by `(task, source)` `[Cx1,Cx5]`. |
| Task-mode tracking | The task's fan-out mode (which sources it has) is tracked so `cancel_task()` cancels all of them, not just the peer `[Cx1]`. |

## 3. Scope

### 3.1 In scope
1. `Event.source: Option<String>` (bridge-core); `RouteTarget::Fanout`; `skill=="fan-out"` route + third Agent Card skill.
2. **N-ary merge coordinator** (`crates/bridge-a2a-inbound/src/fanout.rs`): owns a `Vec<Source>` (each = `{ id: String, stream: Stream<Result<Event,BridgeError>> }`), stamps each `Event` with its source id, merges into the one `mpsc`, runs the per-source state machine (startup-error, disposition-based degrade, terminal), is the **sole** channel sender, and emits the task-terminal `StatusUpdate`.
3. **Task-terminal model:** inbound emits a final `StatusUpdate(Completed\|Failed)` as task end; per-source artifacts set `lastChunk` for artifact-completion only.
4. **2.5 outbound parser update** (`bridge-a2a-outbound/src/client.rs`): end on the terminal `StatusUpdate` (terminal `TaskState`), accumulate **multiple** `artifactUpdate`s (each → `Event::artifact`), not just the first `lastChunk=true`.
5. Source-aware SSE (`event_to_sse`): `metadata["a2a-bridge.source"]` + `artifact.name` when `source` is `Some`; plus emitting the task-terminal status frame.
6. **N-source cancellation**: cancel all sources (immediate Kiro by `SessionId`, latched peer), `(task,source)` guard, task-mode tracking.

### 3.2 Out of scope (→ Increment 3)
>2 peers / multiple remote peers; result reconciliation/voting/dedup; race-for-fastest;
per-source timeouts beyond the existing outbound timeout; configurable partial-failure mode.
(The N-ary *shape* is in 2.6; actually wiring >2 sources arrives with the multi-agent backends.)

### 3.3 Success criteria
- **S3.1 (merge).** A `fan-out` request streams both sources' frames, each `metadata`-labeled by
  source; two labeled artifacts; one final `StatusUpdate(Completed)` as task end. Both run to
  completion.
- **S3.2 (degrade by disposition).** A `Failed`-disposition error on one source → its labeled
  error frame + the survivor's result; task `Completed`. A local `PermissionRequired` on a
  fan-out source is treated as that source failing (labeled error), NOT a task suspend, and
  leaves no orphaned pending permission. All sources failed → task `Failed`.
- **S3.3 (startup-error degrade).** A peer `open_stream` connect/non-2xx failure → labeled peer
  error; Kiro still completes; task `Completed`.
- **S3.4 (cancel-all).** Inbound `CancelTask`/disconnect on a fan-out task fires Kiro
  `session/cancel` (immediately, by `SessionId`) AND peer `CancelTask`, exactly once each;
  cancel before the peer id exists is latched, not blocked on Kiro.
- **S3.5 (terminal model).** Our own outbound parser, pointed at our fan-out output, yields BOTH
  artifacts and ends on the terminal status (proves the round-trip; S3.6 bridge-to-bridge).
- **S3.6 (labeling).** Every frame round-trips as `a2a::StreamResponse` with the right
  `metadata` source; single-source (Local/Delegate) SSE terminal is the same model (a final
  task `StatusUpdate`) — see §5 note on single-source consistency.

## 5. Data flow & semantics

- **Source struct (coordinator-internal):** `Source { id: String, stream: Pin<Box<dyn Stream<Item=Result<Event,BridgeError>>+Send>>, cancel: <handle> }`. 2.6 builds `[kiro, peer]`.
- **Per-source state machine** (each independently):
  - *Startup error* (peer `open_stream` connect/non-2xx, or local spawn failure) → emit one
    labeled error frame (`StatusUpdate`, source id, `TaskState::Failed`), source = failed `[Cx3]`.
  - *Stream item `Ok(ev)`* → stamp source, forward.
  - *Stream item `Err(e)`* → branch on `e.disposition()`: `SetState(Failed)` (or treated-as-failed
    resumable per the policy) → labeled error frame + source = failed; the source stops. Survivors
    continue `[Cx4]`. (Fan-out never suspends per-source; an `InputRequired`/`AuthRequired` on a
    fan-out source is rendered as that source failing — no orphaned pending state.)
  - *Clean end* (stream done, maybe after an artifact, maybe with no artifact `[Cl-minor]`) → source = succeeded.
- **Merge:** `futures::stream::select_all` over the source streams; the coordinator is the only
  task sending to the `mpsc`. **Done-when-all:** ends when every source has terminated.
- **Task terminal:** after all sources terminate, emit one `StatusUpdate(Completed)` if ≥1
  succeeded, else `StatusUpdate(Failed)` (→ the SSE's task end). Per-source artifacts carried
  `lastChunk=true` for *their* artifact, but the task end is this status frame `[Cl4]`.
- **Single-source consistency:** to keep one terminal model, the Local and Delegate single-source
  paths also end with a task-terminal `StatusUpdate` (and the outbound parser ends on it). This
  is the §3.1.4 parser/terminal change; it supersedes 2.5's "end on first `lastChunk`".
- **Cancel:** the coordinator's `select!` (next merged event / `tx.closed()` / `cancel_requested`)
  cancels **all** sources. Kiro is cancelled **immediately** via the known `SessionId` (never
  awaiting a peer id `[Cx5]`); the peer cancel is latched and applied when its id appears. Each
  cancel is `(task, source)`-guarded; an already-terminated source is a no-op.
- **Unary `SendMessage` + fan-out** `[Cl-minor]`: collect ALL frames and return **both** labeled
  artifacts in the response (do NOT collapse to one — that would drop an opinion). (Streaming is
  the primary path; unary returns the full labeled set.)
- **Cancel after one source finished** `[Cl-minor]`: cancel only the still-running sources; the
  finished one's result stands; task terminal reflects the survivors.

## 6. Error model

Reuse `BridgeError`; no new variants. A per-source failure is a labeled `StatusUpdate(Failed)`
frame (the source id in `metadata`), not a `BridgeError` on the merged stream. Only the
all-sources-failed case yields the task-terminal `StatusUpdate(Failed)`. Degrade vs suspend is
decided by `BridgeError::disposition()` (§5); resumable dispositions are treated as source
failure in fan-out (no per-source suspension).

## 7. Testing

- **Coordinator (unit, N-ary, against fake source streams):** all-succeed → labeled artifacts +
  one terminal `Completed`; one source `Failed`-disposition → degrade (survivor + labeled error,
  `Completed`); a source `Err` with `InputRequired` disposition → treated as failure (labeled
  error, no orphaned pending), `Completed`; **startup error** (a source pre-failed) → degrade;
  all fail → terminal `Failed`; a source that ends with **no artifact** → counts as succeeded;
  every frame carries the right source id; done-when-all.
- **Outbound parser (2.5 update):** a stream with **two** `artifactUpdate`s then a terminal
  `StatusUpdate(Completed)` → yields **both** artifacts and ends on the status (not the first
  `lastChunk`); `Failed` terminal → error. Existing single-artifact tests updated to the new
  terminal model.
- **SSE labeling/terminal:** `event_to_sse` with `Some(id)` round-trips as `StreamResponse` with
  `metadata` source + `artifact.name`; the task-terminal `StatusUpdate` is emitted.
- **Inbound integration (CI-safe):** `fan-out` skill → real `InboundServer` (fake local backend +
  mock peer) → merged SSE: two labeled artifacts + terminal status (S3.1); degrade (S3.2);
  startup-error degrade (S3.3).
- **Cancel-all:** inbound `CancelTask`/disconnect on a fan-out task → Kiro cancel (immediate, by
  `SessionId`) AND peer cancel, exactly once each; early cancel (no peer id yet) doesn't block
  Kiro cancel (S3.4).
- **Round-trip (S3.5):** our outbound parser consumes our own fan-out SSE and yields both artifacts.
- **Gated `#[ignore]` e2e (S3.6):** fan-out to real Kiro + a second `a2a-bridge` peer; both
  labeled artifacts return through the now-correct terminal model.
- Coverage gates unchanged (workspace ≥85%; bridge-core/bridge-acp ≥90%).
