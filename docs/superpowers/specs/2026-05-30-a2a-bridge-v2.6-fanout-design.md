# A2A Bridge — Increment 2.6 Design Spec (Fan-out + Concurrent Stream-Merge)

*Date: 2026-05-30*
*Status: Approved design — ready for implementation planning*
*Builds on: v1 + Increment 2.5 (both merged to `main`)*

---

## 1. Purpose

Add a **`fan-out` skill**: one inbound A2A task runs the same prompt on **both the local
Kiro agent and the one configured remote A2A peer** concurrently, and the bridge **merges
their two event streams into the single caller SSE, source-labeled, both running to
completion** (a "second opinion" — the caller gets both agents' answers to compare).

This realizes the concurrent stream-merge deferred from 2.5 (2.5 §8). It is the substance
of 2.6: the merge stage (ordering, per-source labeling, done-when-both, partial-failure)
on top of the building blocks 2.5 shipped (the streaming `DelegationPort`, the peer-id
`watch`/cancel handle, the `mpsc → SSE` path).

## 2. Decisions (locked)

| Decision | Choice |
|----------|--------|
| Semantics | Second opinion: same prompt to Kiro AND the peer; both run to completion; both results returned, source-labeled. NOT race-for-fastest. |
| Sources | Kiro (local) + the one configured `[delegation]` peer. No N-way fan-out, no multiple peers, no result reconciliation/voting (the caller compares). |
| Labeling | Per-frame: `metadata["a2a-bridge.source"]="kiro"|"peer"` on every `StreamResponse` frame; artifact frames also set `artifact.name`. `Event` gains `source: Option<Source>`. |
| Partial failure | **Degrade to survivor**: one source fails → emit the survivor's result + a labeled error frame for the failed source; task `Completed`. Both fail → task `Failed`. |
| Done | The merged stream ends when **both** source streams terminate. |
| Routing | `RouteTarget::Fanout`; `skill=="fan-out"`; Agent Card gains a third skill. |
| Merge point | Reuse the existing `mpsc<Result<Event,BridgeError>> → SSE`; the new merge coordinator runs both producers into it. |

## 3. Scope

### 3.1 In scope
1. **`Event` source tag**: `Event { ..., source: Option<Source> }`, `Source { Local, Peer }`; `None` default (single-source paths unchanged).
2. **`RouteTarget::Fanout`** + `skill=="fan-out"` route + the third Agent Card skill.
3. **Merge coordinator** (`crates/bridge-a2a-inbound/src/fanout.rs`): runs the local
   (`Translator`-over-`KiroBackend`) and peer (`DelegationPort`) producers concurrently,
   stamps each `Event` with its `Source`, merges both into the one `mpsc → SSE`, applies
   degrade-to-survivor, and ends when both terminate.
4. **Source-aware SSE**: `event_to_sse` sets `metadata["a2a-bridge.source"]` + (for artifacts)
   `artifact.name` when `source` is `Some`; unchanged when `None`.
5. **Two-source cancellation**: inbound `CancelTask` / caller-disconnect cancels **both**
   Kiro (`backend.cancel(session)`) and the peer (`delegation.cancel(peer_task)`), each
   guarded against double-cancel.

### 3.2 Out of scope (→ later)
N-way fan-out / >2 sources; multiple peers; result reconciliation/voting/dedup; race-for-
fastest (cancel-loser) semantics; per-source timeouts beyond the existing outbound timeout;
configurable partial-failure mode (degrade is fixed in 2.6).

### 3.3 Success criteria
- **S3.1 (merge).** A `fan-out`-skill `SendStreamingMessage` runs the prompt on both Kiro and
  the peer; the caller's SSE contains BOTH a `source=kiro` artifact and a `source=peer`
  artifact, with interim status frames labeled by source. Task `Completed` when both finish.
- **S3.2 (degrade).** If the peer fails (e.g. `UpstreamA2aError`) while Kiro succeeds, the SSE
  carries Kiro's labeled artifact + a `source=peer` labeled error frame; task `Completed`.
  If both fail, task `Failed`.
- **S3.3 (cancel-both).** An inbound `CancelTask` / caller disconnect on a fan-out task fires
  **both** a Kiro `session/cancel` and a peer `CancelTask`, exactly once each.
- **S3.4 (labeling).** Every merged frame round-trips as `a2a::StreamResponse` and carries the
  correct `metadata["a2a-bridge.source"]`; single-source (Local/Delegate) SSE is byte-unchanged.

## 4. Architecture & component changes

```
caller ──SendStreamingMessage(skill=fan-out)──▶ bridge-a2a-inbound
   gate(): auth → version → route = Fanout
                                   │
                          spawn_fanout_producer  (fanout.rs)
              ┌────────────────────┴────────────────────┐
        local stream                               peer stream
   Translator over KiroBackend                DelegationPort.delegate
        (stamp Source::Local)                   (stamp Source::Peer)
              └──────────── merge (select) ─────────────┘
                       degrade-to-survivor; done-when-both
                                   │
                         mpsc<Result<Event,_>> ──▶ SSE
                         (event_to_sse stamps source metadata + artifact.name)
```

| File | Change |
|------|--------|
| `bridge-core/src/translator.rs` | `Event { source: Option<Source> }` + `Source { Local, Peer }`; `Event::with_source(self, Source)` (or a stamping helper); existing ctors default `source: None`. |
| `bridge-core/src/domain.rs` | (if `Source` lives here) `Source` enum + `RouteTarget::Fanout`. |
| `bridge-a2a-inbound/src/fanout.rs` (new) | `spawn_fanout_producer(...)`: build + stamp + merge the two source streams; degrade logic; done-when-both; the two-source cancel supervisor. |
| `bridge-a2a-inbound/src/server.rs` | `gate()` recognizes `Fanout`; the streaming/unary handlers dispatch `Fanout → fanout::spawn_fanout_producer` into the existing `mpsc → SSE`; `cancel_task()` cancels both sources for a fan-out task. |
| `bridge-a2a-inbound/src/sse.rs` | `event_to_sse` reads `Event.source` → sets `metadata["a2a-bridge.source"]` + `artifact.name` when `Some`. |
| `bridge-a2a-inbound/src/card.rs` | advertise the third `fan-out` skill. |
| `bin/.../route.rs` | `SkillRoute`: `skill=="fan-out"` → `RouteTarget::Fanout`. |
| `bridge-store` | reuse the existing `local→peer` mapping + cancel latch; fan-out also needs the local session id reachable for Kiro cancel (the local producer already manages a session — surface its id to the coordinator's cancel path). |

## 5. Data flow & semantics

- **Stamping:** the coordinator wraps each source's `Stream<Result<Event,_>>` and maps every
  `Ok(ev)` to `ev.with_source(Local|Peer)`. Errors are handled per the degrade rule below.
- **Merge:** `futures::stream::select` (or equivalent) over the two stamped streams feeds the
  one `mpsc`. Done-when-both is inherent: the merged stream ends only when both inner streams
  end. Ordering is arrival-interleaved; the per-frame `source` preserves attribution.
- **Degrade-to-survivor:** a per-source `Err(e)` is caught inside the coordinator and converted
  to a labeled error `Event::status(<e>)` with that source (NOT a task-terminal error); that
  source's stream is then complete. The survivor continues. Track per-source success/failure.
- **Terminal:** when both sources have terminated — if **both failed**, the coordinator sends a
  single final `Err(BridgeError::UpstreamA2aError)` (→ task `Failed`); else the stream ends
  cleanly (→ task `Completed`), both/either labeled artifacts already delivered.
- **Cancel:** the coordinator's `select!` (next merged event / `tx.closed()` / `cancel_requested`)
  cancels BOTH sources on disconnect or inbound `CancelTask`: `backend.cancel(local_session)`
  and `delegation.cancel(peer_task)` (await the peer-id `watch` / honor the early-cancel latch).
  Extend 2.5's single-cancel guard to a per-source guard keyed by `(task, source)` so neither
  source is double-cancelled; a source that already terminated is a cancel no-op.

## 6. Error model

Reuse `BridgeError`. A per-source failure is rendered as a labeled error `Status` frame, not a
`BridgeError` on the merged stream (degrade). Only the both-failed terminal emits a
`BridgeError` (→ `Failed` via `disposition()`). No new variants.

## 7. Testing

- **Merge coordinator (unit, isolation against two fake `Stream<Result<Event,_>>`):**
  both-succeed → two `source`-labeled artifacts + done-when-both; one-source-`Err` → degrade
  (survivor artifact + a labeled error frame, task `Completed`); both-`Err` → final `Err`
  (task `Failed`); every emitted frame carries the correct `source`.
- **SSE labeling (`sse.rs`):** `event_to_sse` with `Some(Local|Peer)` round-trips as
  `a2a::StreamResponse` carrying `metadata["a2a-bridge.source"]` + (artifacts) `artifact.name`;
  `None` is byte-unchanged vs today.
- **Inbound integration (CI-safe):** `fan-out`-skill request → real `InboundServer` with a fake
  local backend + a mock peer → merged SSE shows both sources' labeled artifacts (S3.1) and the
  degrade case (S3.2).
- **Cancel-both:** inbound `CancelTask` / caller-disconnect on a fan-out task → both
  `backend.cancel` and `delegation.cancel` fire exactly once each (S3.3).
- **Gated `#[ignore]` e2e:** fan-out to real Kiro + a second `a2a-bridge` instance as the peer;
  assert both labeled artifacts return.
- Coverage gates unchanged (workspace ≥85%; bridge-core/bridge-acp ≥90%).

## 8. Forward note

After 2.6, the natural next steps are **Increment 3** (multi-agent adapters — Claude
Code/Codex/Gemini — which would generalize fan-out beyond Kiro+peer to N sources, and the
conductor fork/no-fork re-evaluation) and wiring the `agent-client-protocol` SDK. N-way
fan-out / multiple peers / result reconciliation are deliberately deferred until the
multi-agent backend surface exists.
