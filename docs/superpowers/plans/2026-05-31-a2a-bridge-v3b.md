# A2A Bridge Increment 3b — Agent Registry Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single hardcoded local backend with a runtime-mutable registry of named agent entries, selected by a dedicated `agent` axis (+ default + raw override), dispatched by a meaningful `AgentId`, reconciled live from a `ConfigSource` (File adapter + hot-reload) — agents add/edit/removable without restart, including live model edits.

**Architecture:** A `ConfigSource` port (File adapter) yields a `RegistrySnapshot`; a reconciler atomically diffs it into an `AgentRegistry` (`ArcSwap<RegistryState{slots,default}>`; slots hold `ArcSwap<AgentEntry>` + lazy `OnceCell<backend>` + `retired`/`leases`). Dispatch `resolve`s `(entry, backend, lease)`, **always** applies the entry's current effective config per-session via `AgentBackend::configure_session` (stashed per `SessionId`, applied at lazy ACP mint), and **instance-binds** the task so multi-message follow-ups reach the same backend. Edit/remove retire the old backend instance via an async `retire()` after leases drain.

**Tech Stack:** Rust 2021 (1.94), `tokio`, `arc-swap`, `notify`, `agent-client-protocol` =0.12.1, `serde`/`toml`, `a2a-lf` =0.3.0.

**Spec:** `docs/superpowers/specs/2026-05-31-a2a-bridge-v3b-design.md` (rev3, dual-reviewed). Section tags `[§N]` reference it; `[Cx]/[Cl]` trace accepted review findings.

**Branch:** create `feat/v3b-registry` off `main` before Task 1.

> **Plan rev2 — dual plan-review folded** (Codex gpt-5.5 + Claude opus-4.8): `forget_session` is a trait method (T2); `resolve` uses an iterative loop with the lease acquired *before* the retired-check (closes the retirement race, avoids recursion-boxing) (T3); `Registry::new` validates → boot fails loud (T3/T12); `apply()` snippet fixed (no move-before-use) (T4); `retire()`/`forget_session` compile against the real `Option<AcpConfig>` + `Arc<StdMutex<Option<Supervised>>>` fields (T6); `task_meta_from_params` is fallible and rejects invalid agent/effort (T9); the `backend`→`registry` migration covers ALL ~6 `srv.backend` sites and `RoutedCall` carries overrides + selected agent, fan-out holds its resolved instance (T10); follow-ups check the binding *before* routing and an RAII guard evicts the lease/stash on *every* producer exit (T11); deps (`futures`/`notify`/`tempfile`/`bridge-registry`) added (T0). Reviewers confirmed the tricky parts correct: tokio `OnceCell`, the `#[async_trait]` boxing, the per-session stash keying (`ensure_session`'s `key` IS the bridge `SessionId`), and retire-by-`ptr_eq`.

```rust
struct Slot { entry: ArcSwap<AgentEntry>, backend: tokio::sync::OnceCell<Arc<dyn AgentBackend>>,
              retired: AtomicBool, leases: Arc<AtomicUsize> }
struct State { slots: HashMap<AgentId, Arc<Slot>>, default: AgentId }
pub struct Registry { state: ArcSwap<State>, spawn: SpawnFn }
struct LeaseGuard(Arc<AtomicUsize>);
impl Drop for LeaseGuard { fn drop(&mut self){ self.0.fetch_sub(1, SeqCst); } }
impl bridge_core::ports::Lease for LeaseGuard {}
```

`resolve(id)` — **iterative loop** (not recursion — avoids the `async_trait` recursion-boxing question `[Cx-B1]`), and **acquire the lease BEFORE the retired-check** to close the race where retirement sees `leases==0` between the check and the increment `[Cx-B1/Cl]`:
```
loop {
    let slot = { let st = self.state.load(); st.slots.get(id).cloned() }   // clone Arc; guard dropped before await [Cl]
        .ok_or(BridgeError::UnknownAgent{ id: id.as_str().into() })?;
    let backend = slot.backend
        .get_or_try_init(|| (self.spawn)(slot.entry.load_full())).await?.clone();
    let lease = LeaseGuard(slot.leases.clone());                 // fetch_add(1) in LeaseGuard::new — BEFORE the retired check
    if slot.retired.load(SeqCst) {                               // spawn-race / concurrent retire [Cx-H4/Cl]
        drop(lease);                                             // give the lease back so retirement can drain
        let _ = backend.retire().await;
        continue;                                                // re-resolve against current state (retired slots leave the map → bounded)
    }
    return Ok(Resolved{ entry: slot.entry.load_full(), backend, lease: Box::new(lease) });
}
```
`LeaseGuard::new` does `leases.fetch_add(1, SeqCst)`; `Drop` does `fetch_sub`. `default_id()` = `self.state.load().default.clone()`; `list()` = keys.
**`Registry::new(snapshot, spawn) -> Result<Self, BridgeError>`** `[Cx-H4]`: validates the snapshot (the SAME `validate()` used by `apply()`, T4 — unique ids, cmds ∈ allowed_cmds, default resolves) so **malformed boot config fails loudly (spec §7)**; returns `ConfigInvalid` on failure. Factor `validate(&RegistrySnapshot)` as a free fn used by both `new` and `apply`.
