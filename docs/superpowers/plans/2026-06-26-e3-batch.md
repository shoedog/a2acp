# E3 — Parallel Batch Dispatch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `a2a-bridge run-batch <workflow> --manifest <file>` submits N independent workflow runs to a running `serve`, runs them concurrently under a serve-wide concurrency cap, durably — each child a normal detached task (W3b-resumable, E1-isolated, cancelable), a durable `batch` parent record giving one-query status, cancel-the-batch, and crash-safe tail-resume.

**Architecture:** A bounded-concurrency batch layered over the existing `spawn_detached_workflow` detached primitive. A serve-wide `Semaphore` (in an `Option<BatchRuntime>` owned by `InboundServer` for serve + `Coordinator` for mcp) caps concurrency across all batches. The admission loop + resume + roll-up are **free functions in `bridge-coordinator::batch` over a `BatchDeps` bundle** (mirroring `DetachedDeps`), called from both the inbound-server and coordinator sides. A durable `batch` table + `tasks.{batch_id,item_id}` (additive migration) holds the plan; a `claim_batch_child` store primitive owns spawn-ownership.

**Tech Stack:** Rust, tokio (`Semaphore`/`FuturesUnordered`/`CancellationToken`), rusqlite (additive migration), the existing W3a/W3b detached substrate.

**Binding spec:** `docs/superpowers/specs/2026-06-26-e3-batch.md` — the **`## v3` (RR-FIX-1..14, D11–D14)** + **`## v2` (SR-FIX-1..17)** sections supersede the v1 body. Base = `main` `80be1b4`. Branch `feat/e3-batch` (already created; spec committed `7afc526`).

use bridge_core::task_store::{BatchRecord, BatchStatus, BatchSummary, TaskRecord, TaskRecordStatus};

/// Pure roll-up over the durable plan (rec.total) + the child rows. The SINGLE owner
/// of the bucket math (RR-FIX-7) — never re-implemented in a store impl.
pub fn summarize_batch(rec: &BatchRecord, children: &[TaskRecord]) -> BatchSummary {
    let mut ok = 0; let mut failed = 0; let mut canceled = 0; let mut running = 0;
    let mut kids = Vec::with_capacity(children.len());
    for c in children {
        match c.status {
            TaskRecordStatus::Completed => ok += 1,
            TaskRecordStatus::Failed | TaskRecordStatus::Interrupted => failed += 1,
            TaskRecordStatus::Canceled => canceled += 1,
            TaskRecordStatus::Working => running += 1,
        }
        kids.push((c.item_id.clone().unwrap_or_default(), c.id.clone(), c.status));
    }
    let pending = rec.total.saturating_sub(children.len() as u32);
    BatchSummary { id: rec.id.clone(), workflow: rec.workflow.clone(), status: rec.status,
        total: rec.total, ok, failed, canceled, running, pending, children: kids }
}

/// The SINGLE settle predicate (RR-FIX-8): Working drained → Completed; Canceling with
/// no running child → Canceled. Returns the terminal to CAS into, or None.
pub fn is_settleable(s: &BatchSummary) -> Option<BatchStatus> {
    match s.status {
        BatchStatus::Working if s.pending == 0 && s.running == 0 => Some(BatchStatus::Completed),
        BatchStatus::Canceling if s.running == 0 => Some(BatchStatus::Canceled),
        _ => None,
    }
}
```

/// Validate + persist + spawn the admission loop; return the BatchId. Server-side cwd
/// validation (RR-FIX-13) + zero/empty/dup rejects + None-runtime reject (RR-FIX-6).
pub async fn run_batch(deps: &BatchDeps, params: BatchParams) -> Result<bridge_core::ids::BatchId, BridgeError> {
    // 1. reject if workflow unknown (deps.detached.workflows.get)
    // 2. reject empty items / dup item_id / per-item session_cwd outside allowed_cwd_root
    //    (validate_cwd(s, root, "batch.item.cwd"))  ... ConfigInvalid{reason: item_id} on item failure
    // 3. concurrency = min(params.concurrency.unwrap_or(default_concurrency), max_concurrent)
    // 4. create_batch(Working, items_json={"v":1,"items":[…]})
    // 5. register a CancellationToken in runtime.batch_cancels[bid]
    // 6. tokio::spawn(run_batch_admission(deps.clone(), bid.clone(), items, concurrency, token));
    // 7. return bid
}

async fn run_batch_admission(deps: BatchDeps, bid: BatchId, items: Vec<BatchItem>, concurrency: u32, token: CancellationToken) {
    let mut pending: VecDeque<BatchItem> = items.into();
    let mut inflight: FuturesUnordered<JoinHandle<()>> = FuturesUnordered::new();
    loop {
        // admit while permits + slots + work remain and NOT canceling:
        while inflight.len() < concurrency as usize && !pending.is_empty() && !token.is_cancelled() {
            let item = pending.front().cloned().unwrap();
            // RR-FIX-1: biased select cancel vs acquire — cancel never creates a row.
            let permit = tokio::select! {
                biased;
                _ = token.cancelled() => break,
                p = deps.runtime.semaphore.clone().acquire_owned() => p.expect("semaphore not closed"),
            };
            pending.pop_front();
            let task = new_detached_task_id();
            let rec = build_child_record(&deps, &bid, &item, &task); // Working, batch_id, item_id, workflow_spec_json
            match deps.detached.task_store.claim_batch_child(&bid, &item.item_id, &rec).await {
                Ok(ChildClaim::Created) => {
                    // RR-FIX-2 re-check AFTER row+token: if the batch flipped Canceling, finalize the row.
                    if batch_is_canceling(&deps, &bid).await {
                        let _ = finalize_detached(&deps.detached.task_store, &deps.detached.progress_hubs,
                            &task, TaskRecordStatus::Canceled, None, None, None).await;
                        drop(permit); continue;
                    }
                    let hub = register_hub(&deps, &task);
                    let child_tok = register_child_cancel(&deps, &task);
                    let h = spawn_detached_workflow(&deps.detached, task, item.input, graph(&deps,&bid),
                        run_id, child_tok, HashMap::new(), ctx(&item), hub);
                    inflight.push(carry_permit(h, permit)); // permit drops when the child completes
                }
                Ok(ChildClaim::ExistingWorking) | Ok(ChildClaim::ExistingTerminal) => { drop(permit); }
                Err(_) => { drop(permit); /* leave pending? no — count as failed item, continue */ }
            }
        }
        if inflight.is_empty() && (pending.is_empty() || token.is_cancelled()) { break; }
        tokio::select! {
            biased;
            _ = token.cancelled() => { /* RR-FIX-9: loop is the sole canceller */ cancel_inflight_children(&deps).await; }
            _ = inflight.next() => { /* a child finished → its permit dropped; loop re-admits */ }
        }
    }
    // RR-FIX-8: settle via the single predicate.
    settle_now(&deps, &bid).await; // summarize_batch → is_settleable → settle_batch_if_status / cancel→canceled
    deps.runtime.batch_cancels.lock().await.remove(&bid);
}
```

Key invariants to honor (the tests pin them): the permit is acquired BEFORE `claim`/spawn and released ON child completion (carried into the JoinHandle); `cancel_batch` (T8) CAS-flips `working→canceling` AND fires the batch token, so the loop both stops admitting and (RR-FIX-9) is the sole canceller of in-flight children it owns; a `Created`-then-`Canceling` race finalizes the row durably (RR-FIX-2).
