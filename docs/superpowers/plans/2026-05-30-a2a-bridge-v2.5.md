# A2A Bridge Increment 2.5 (Outbound Delegation, Passthrough) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Forward a single inbound A2A task wholesale to one configured remote A2A peer and stream the peer's events back to the caller (realizing v1's deferred S2), plus thread real message content end-to-end.

**Architecture:** A streaming `DelegationPort` (in `bridge-core`) is implemented by a hand-rolled `reqwest` + `a2a-lf` outbound client (`bridge-a2a-outbound`). The inbound server routes `skill="delegate"` to the `DelegationPort` producer feeding the *existing* `mpsc → SSE` path; `Part` gains text content forwarded to both the peer and Kiro. Completion is an A2A terminal signal, not socket EOF.

**Tech Stack:** Rust 2021 (toolchain 1.94.0), `tokio`, `reqwest` (json+stream+rustls-tls), `a2a` (package `a2a-lf` =0.3.0) wire types, `axum` (mock peer in tests), `serde_json`, `tokio-stream`/`futures`, `async-stream`.

**Spec:** `docs/superpowers/specs/2026-05-30-a2a-bridge-v2.5-outbound-design.md`. Tags `[Codex N]`/`[Claude]` trace to accepted spec-review findings.

**Branch:** create `feat/v2.5-outbound` off `main` before Task 1.
