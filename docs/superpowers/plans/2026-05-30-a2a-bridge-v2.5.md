# A2A Bridge Increment 2.5 (Outbound Delegation, Passthrough) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Forward a single inbound A2A task wholesale to one configured remote A2A peer and stream the peer's events back (realizing v1's deferred S2); thread real message content end-to-end; and make inbound SSE A2A-conformant so inbound/outbound are symmetric.

**Architecture:** A streaming `DelegationPort` (in `bridge-core`) implemented by a hand-rolled `reqwest` + `a2a-lf` outbound client. The inbound server routes `skill="delegate"` to the `DelegationPort` producer feeding the existing `mpsc → SSE` path; inbound SSE is upgraded to emit real `a2a::StreamResponse`. `Part` gains text content forwarded to peer and Kiro. Completion = A2A terminal signal, not socket EOF.

**Tech Stack:** Rust 2021 (1.94.0), `tokio`, `reqwest` (json+stream+rustls-tls), `a2a` (package `a2a-lf` =0.3.0), `axum` (mock peer in tests), `serde_json`, `tokio-stream`/`futures`, `async-stream`.

**Spec:** `docs/superpowers/specs/2026-05-30-a2a-bridge-v2.5-outbound-design.md`. Tags `[Cx]`/`[Cl]` trace to accepted plan-review findings (Codex/Claude, 2026-05-30).

**Branch:** `feat/v2.5-outbound` (already created).
