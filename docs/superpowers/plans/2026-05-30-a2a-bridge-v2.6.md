# A2A Bridge Increment 2.6 (Fan-out + Concurrent Stream-Merge) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A `fan-out` skill that runs one task on both Kiro and the configured peer concurrently and merges their event streams into one source-labeled SSE (second opinion), with an N-ary coordinator, a correct A2A task-terminal model, degrade-to-survivor, and two-source cancellation.

**Architecture:** A new N-ary merge coordinator (`fanout.rs`) owns a `Vec<Source>` (each = id + event stream + cancel handle), stamps each `Event` with its source, merges via `select_all` into the existing `mpsc → SSE`, and emits one `Event::terminal(outcome)` after all sources end. `Event` gains a `source` label and a `Terminal` kind; the terminal model (a final `StatusUpdate`) is unified across single-source and fan-out paths, and the 2.5 outbound parser is updated to match.

**Tech Stack:** Rust 2021 (1.94.0), `tokio`, `futures` (`stream::select_all`), `a2a` (`a2a-lf` 0.3.0), `axum`, `reqwest`, `serde_json`.

**Spec:** `docs/superpowers/specs/2026-05-30-a2a-bridge-v2.6-fanout-design.md`. Tags `[Cx]`/`[Cl]` trace to accepted spec-review findings.

**Branch:** create `feat/v2.6-fanout` off `main` before Task 1.
