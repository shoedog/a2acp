**BLOCKER — `src/lib.rs:29–30`, `Upstream` arm leaks internal detail onto the wire (WRONG).** In `client_message()` the `Upstream` arm does `format!("upstream request failed: {self}")`, and `{self}` invokes `Display`, whose `Upstream` arm (`src/lib.rs:15`) writes `"upstream {url} failed: {source}"`. So a client receives e.g. `"upstream request failed: upstream http://internal-agent:8080/v1 failed: connection refused"`, exposing the upstream URL (host/topology) and the raw source error. This directly violates the documented no-internal-detail contract at `src/lib.rs:23–25` — the exact leak the change was meant to prevent. **Fix:** return a static category with no field interpolation, e.g. `"upstream request failed".to_string()`; keep `url`/`source` only in the `Display`/log path. (Both reviewers independently agree; this is the sole blocker.)

**MAJOR — the wire boundary is unenforced by construction; nothing stops the next variant repeating this (architecture root cause).** `client_message` returns `String`, which invites `format!`/interpolation of variant fields, so correctness depends on every future author remembering the rule rather than the type system enforcing it. This is also why no test caught the regression. **Fix:** make the wire side structurally incapable of carrying payload — return `&'static str` (or a closed `Category` enum mapped to fixed strings) so a leak like the BLOCKER cannot compile; and add a boundary test that constructs `Upstream { url: "SECRET_HOST", source: "SECRET_SRC" }` and asserts `client_message()` equals the generic category and contains neither substring. *(Codex and Claude both flagged the missing test as MAJOR/MINOR; folded here since the structural fix and the test together lock the invariant.)*

**MINOR — `source: String` pre-flattens the error, losing the chain and typed category (`src/lib.rs:6`, smell).** Storing the cause as an already-formatted string means the boundary can never distinguish upstream-timeout vs. auth-rejected vs. 5xx without re-parsing text, and structured logging can't reconstruct the error chain. **Fix:** hold a typed source (`#[source] Box<dyn Error + Send + Sync>` or an enum) and derive both the log string and the wire category from it. *(Claude-only; Codex did not surface it, but it's a genuine forward-looking design gap, not a correctness defect.)*

**Overall verdict:** Do not merge — fix the 1 BLOCKER (`{self}` interpolation); land the MAJOR (typed `&'static str`/category wire boundary + secrecy test) alongside it so the leak becomes impossible by construction rather than by convention.
