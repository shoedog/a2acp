**Merged Review: rs-06 — `EventLog::events_since`**

**BLOCKER — `src/lib.rs:31`, off-by-one boundary predicate.** The filter uses `e.seq >= cursor`, which replays the event whose `seq == cursor` — but that event was the subscriber's `Last-Event-ID` and was already delivered. Concrete failure: a subscriber caught up through seq 5 reconnects with `Last-Event-ID: 5`, calls `events_since(5)`, and gets `[5, 6, 7, …]`, re-delivering event 5. This breaks the exactly-once-across-reconnect guarantee the method exists to provide. Fix: `e.seq > cursor`. (Both reviewers flagged this; it is the central defect. Codex tagged it MAJOR/WRONG, Claude BLOCKER — Claude's severity is right: an off-by-one that violates the sole documented correctness contract of the change is release-blocking.)

**MAJOR — `src/lib.rs:29-33`, O(n) scan discards the `index == seq - 1` invariant.** `append` sets `seq = self.events.len() + 1`, so seq is 1-based, contiguous, and monotonic; the first event to replay sits at vector index exactly `cursor`. The linear `filter` re-scans every event on every reconnect (costly on a hot, large log) and silently tolerates seq gaps that `append` guarantees cannot exist — encoding a weaker assumption than the type upholds. Fix: `self.events.get(cursor as usize..).unwrap_or(&[])` — O(1) locate plus one tail copy. (Claude only; a legitimate correctness-adjacent efficiency point grounded in the fixture's invariant.)

**MAJOR (design) — `src/lib.rs:28-34`, the reattach invariant has no seam.** The `> cursor` boundary *is* the entire correctness contract, yet it lives as an anonymous inline closure in one method. Add a second replay path (a bounded replay window, or `events_between(lo, hi)` for paging) and this boundary gets hand-re-implemented and drifts — the same off-by-one recurs. Direction: extract a single `first_index_after(cursor) -> usize` helper that encodes the boundary once, and route every replay path through it. (Claude only.)

**MINOR — `src/lib.rs:28`, returns owned `Vec<Event>` (deep clone).** Every reattach forces a full clone of the missed span regardless of caller need; the SSE caller typically just serializes the tail. Returning `&[Event]` (or `impl Iterator<Item = &Event>`) lets the caller decide whether to clone. (Claude only.)

**MINOR — overrun/impossible cursor is indistinguishable from "caught up."** If `cursor` exceeds the head seq (a `Last-Event-ID` the server never emitted, or a log that was reset/truncated), the method returns empty — identical to the healthy "nothing missed" case, so a durable-replay client silently stalls with a permanent gap instead of resuming. Direction: a typed result (`UpToDate | Replay(..) | CursorAhead`), or at minimum a documented precondition and a log on overrun. (Claude only.)

**MINOR — the change's single most critical invariant ships untested.** There is no tests module in the fixture, so the `seq == cursor` boundary is unasserted. Both reviewers raised this; merge to one action: add tests for `cursor = 0` → all events, `cursor = existing seq` → excludes that seq (e.g. append 1..=3, assert `events_since(2)` yields only seq 3), and `cursor > last seq` → empty. Any one of these would have caught the BLOCKER at authoring time and guards against the recurrence noted in the design finding above.

**Verdict:** Reject — ship only after fixing the 1 BLOCKER (`>=` → `>`); strongly recommend the index-based slice and a boundary test in the same change, since both directly harden the invariant that was violated.
