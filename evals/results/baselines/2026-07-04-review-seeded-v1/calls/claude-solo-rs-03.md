**BLOCKER — `client_message()` leaks internal detail through `{self}` (src/lib.rs:29–31).** The `Upstream` arm interpolates `{self}`, which invokes `Display` (line 14–16: `"upstream {url} failed: {source}"`). So the wire message becomes `"upstream request failed: upstream https://internal-host/... failed: <source error>"` — the exact upstream URL, host topology, and underlying source error the doc comment on line 23–25 forbids. This is a defect (wrong, not smell): the concrete input `Upstream { url: "http://10.0.0.4:8080/agent", source: "connection refused" }` produces a client message disclosing internal infrastructure to an untrusted reader. Fix: return a static category string like the other two arms — `"upstream request failed".to_string()` — and never route wire text through `Display`.

**MAJOR — Hidden coupling: `Display` and `client_message` share one code path with opposite contracts.** `Display` is deliberately the *internal/log-detail* view; `client_message` is the *redacted wire* view. By having the wire arm call `{self}`, the two are fused. The structural consequence: any future edit that enriches `Display` for logging — adding a request id, a retry count, a header dump — silently widens the wire leak with no signal at the leak site. The invariant "wire text is decoupled from Display" isn't enforced anywhere; it survives only by each arm being disciplined, and this arm broke it. Fix direction: treat the two as separate seams and never let one call the other.

**MAJOR — Missing invariant guard; the type signature permits the leak.** Every legitimate client message here is a compile-time constant, yet the method returns `String`, which is what makes `format!("... {self}")` expressible at all. Because there is no structural boundary, "does this arm leak?" is a per-arm manual audit forever — and it will be re-litigated every time a variant is added. Fix direction: make the safe shape the only shape — return `&'static str` (all current messages already are static), forcing any genuinely dynamic wire content through an explicit, reviewable redaction helper rather than free-form interpolation. That converts this whole bug class from "hope the reviewer catches it" into a type error.

**MINOR — `source: String` collapses the error chain (src/lib.rs:6).** Storing the cause as `String` rather than `Box<dyn Error + Send + Sync>` (and the type not implementing `Error::source()`) means downstream `anyhow` context, structured logging, or any consumer that walks `.source()` sees a flattened, unstructured cause. Consequence: the moment upstream failures need programmatic discrimination (retryable vs. fatal) or a real cause chain in logs, this variant has to be reshaped and every construction site touched. Smell, not wrong — but it's the kind of stringly-typed decision that's cheap now and expensive to reverse later.

**Verdict: REJECT** — the `Upstream` arm violates the method's own documented no-leak contract, disclosing upstream URL and source error on the wire; the deeper fix is to decouple wire text from `Display` and make the leak unrepresentable in the type.
