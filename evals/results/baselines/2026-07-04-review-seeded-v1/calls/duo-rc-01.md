**MAJOR**
- **Unbounded accumulation** — `src/lib.rs`, delta arm (`acc.push_str(&d)`). SMELL. `acc` has no cap and no backpressure. A long-running or runaway agent that streams forever without being cancelled grows a single `String` without bound on this warm producer → memory blowup. Note the internal tension: the doc comment cites "a chatty agent" to justify `biased;` for cancel-starvation, yet that same chatty agent is unbounded on the memory axis. Fix: bound accumulation (cap + truncate-or-error), or stream deltas to a sink incrementally instead of buffering the whole reply. (Claude)

**MINOR**
- **No test coverage for the contract** — `src/lib.rs:16` `run_turn`. SMELL. No `#[test]`/`#[tokio::test]` exists, so a future regression (e.g. treating `cancel.recv() == None` as a cancel, or draining a ready delta ahead of an already-queued cancel) would go uncaught. Fix: add async tests for (a) queued cancel vs. ready delta, (b) dropped cancel sender → drains to completion, (c) delta EOF → `Completed`, (d) accumulated multi-delta output. This test should also explicitly pin the same-poll race below. (Codex + Claude — merged: Codex's coverage gap subsumes Claude's "pin this with a test.")
- **Same-poll cancel + delta-EOF race is undocumented** — `src/lib.rs`, `select!`. SMELL. When `Some(())` and `deltas.recv() == None` become ready in the same poll, `biased;` resolves to `Cancelled`, discarding a reply that had in fact just completed. This is consistent with the stated abort-first rule, but the "answer finished AND a cancel raced in" case may surprise callers. Fix: make it an explicit, tested decision (covered by the test above) rather than emergent behavior. (Claude)
- **`Cancelled` discards accumulated work** — `src/lib.rs`, `TurnOutcome::Cancelled` (unit variant). SMELL. On cancel, `acc` is dropped. A likely future warm-turn requirement — "return the partial reply when interrupted" — would force a breaking enum change. Fix: adopt `Cancelled { partial: String }` now, before callers depend on the unit shape. (Claude)
- **Impoverished cancel signal** — `src/lib.rs`, `cancel: Receiver<()>` + reasonless `Cancelled`. SMELL. A `()` payload plus a reasonless outcome cannot express *why* a turn was cancelled; this system has multiple cancel producers (user cancel vs. watchdog idle/wall). Attributing or logging cause later requires changing channel payload and outcome together. Fix: `Receiver<CancelReason>` → `Cancelled(CancelReason)`. (Claude)
- **Accumulate-policy fused to a concrete transport** — `src/lib.rs`, `run_turn` signature. SMELL. The driver couples its accumulate logic to `tokio::sync::mpsc::Receiver` with `()`/`String` payloads. Acceptable at this size, but swapping the delta source to a `Stream`/`broadcast` later means rewriting the body, not just the signature. Low priority. (Claude)

**Overall verdict:** No correctness defect — ship as-is if this is a throwaway fixture; for production, ship after bounding `acc` (the one MAJOR) and adding the contract tests, then fold in the outcome/cancel-signal shape changes before callers lock in the current enum.
